UX Design Basics

UX Design

User experience design is the process of enhancing user satisfaction with a product by improving the usability, accessibility, and satisfaction provided in the interaction with the product. The same way the internet connects us, there are web languages that work together to make up the internet and the websites it contains. HTML, CSS and JavaScript work together to form the front-end design of a website by applying information that affects content, style and interactivity of a site.

Being a UX Designer means you need to demonstrate your design skills visually, and knowing front-end web development can be a part of that process. The basics of front-end web development are HTML, CSS, and JavaScript code. This code controls what a user sees and interacts with on a website. Since front-end developers code websites using the building blocks of HTML, CSS, and JavaScript, having a good understanding of front-end coding can help you communicate better with developers. Additionally, it can potentially eliminate time spent making revisions. By developing a basic understanding of how to code, you are cognizant of the impact your decisions are making on your designs, and how interactions are meaningful for the end user. In the end, it can speed up the product development process.

HTML

Short for HyperText Markup Language, HTML is the language used to create documents on the World Wide Web. HTML defines the structure and layout of a Web document by using a variety of tags and attributes.

Some HTML Element Definitions

  • The <s!DOCTYPE html> declaration defines this document to be HTML5
  • The <HTML> element is the root element of an HTML page
  • The <head> element contains meta information about the document
  • The <title> element specifies a title for the document
  • The <meta> element should define the character set to be UTF-8
  • The <meta> element with name="viewport" makes the website look good on all devices and screen resolutions
  • The <style> element contains the styles for the website (layout/design)
  • The <body> element contains the visible page content
  • The <h1> element defines a large heading
  • The <p> element defines a paragraph

    • CSS

    CSS is essentially like the drapes to your window. You've laid the groundwork for the window frame with html, and CSS comes along to add some panache to your room.

    While HTML is used to create the actual content of the page, such as written text - like you are reading now - CSS (cascading style sheets) is responsible for the design or style of the website, which can describe the presentation of a web page, including colors, layout, and fonts.

    CSS can be added to HTML elements in 3 ways:

  • Inline by using the style attribute in HTML elements
  • Internally by using a <style> element in the <head>
    • section
  • Externally by using an external CSS file

    • An inline CSS is used to apply a unique style to a single HTML element. An inline CSS uses the style attribute of an HTML element.

    This example sets the text color of the <h1> element to red: 
     <h1 style style="color:red;">This is a Red Heading<h1/>

    The result is:

    This is a Red Heading

    An internal CSS is used to define a style for a single HTML page. An internal CSS is defined in the <head> section of an HTML page, within a <style> element:

        
      
<html>
<head>
<style>
body {background-color: powderblue;}
h1   {color: blue;}
p    {color: red;}
</style>
</head>
<body>

<h1>This is a heading</h1>
<p>This is a paragraph.</p>

</body>
</html>

    With an external style sheet, you can change the look of an entire web site, by changing one file! To use an external style sheet, add a link to it in the <head> section of the HTML page, like this:

        
      
     <link rel="stylesheet" href="styles.css">

    Responsive Web Design

    Responsive web design is an approach to web design that makes web pages render well on a variety of devices and window or screen sizes.

    These days we use any number of computers, phones, and tablets to look at web pages. CSS allows a designer to adapt the presentation to different types of devices, such as large computer monitors or small handheld devices like smart phones.

    In this case, media queries are useful when you want to modify your site or app depending upon a device's general type or specific characteristics and parameters. The media query below adjusts parameters for browser viewport width.

    Adjusting design for the width of the viewport can be executed with the following media queries. Additionally, many media features are range features, which means they can be prefixed with "min-" or "max-" to express "minimum condition" or "maximum condition" constraints.

    Media Queries in Action

    Using a min-width media query is designing with a mobile first approach.

        
      
body {
    /* default styles here, 
       targets mobile first */
}

@media screen and (min-width:480px) {
    /* style changes when the screen gets larger */
}

@media screen and (min-width:800px) {
        /* And even larger */
    }

    Whereas a max-width media query would be used if the sites you are working on are designed for desktop first, then adds queries to make styles mobile-friendly.

        
      


body {
        /* default styles here, 
           targets desktops first */
    }

@media screen and (max-width:800px) {
    /* style changes when the screen gets smaller */
}

@media screen and (max-width:480px) {
        /* And even smaller */
    }

    If you want your styles to affect specific criteria, you can just combine the two like below:

     body {
        /* default styles here, 
           targets mobile first
           ALSO will cover 361 - 479 width */
    }

@media screen and (max-width:360px) {
    /* style ONLY for screens with 360px or less width */
}

@media screen and (min-width:480px) {
    /* style changes when the screen gets larger */
       }
